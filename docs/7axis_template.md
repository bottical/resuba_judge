

# レスバ判定テンプレート（7軸評価フレーム）

## 0. 目的

1対1の対立的な議論（レスバ）について、  
**内容（論理・データ）と態度（詭弁・誠実性）を分離して評価**し、  
最終的に「どちらがより妥当で一貫した議論を行っているか」を判断するためのテンプレート。

---

## 1. 入力フォーマット

### 1-1. メタ情報

- 評価対象：
  - Aサイド：{{Aの名前 or ID}}
  - Bサイド：{{Bの名前 or ID}}
- 議論テーマ：{{テーマ要約（例：『普通』の基準／23%データの解釈など）}}
- 対象期間：{{開始日時}} 〜 {{終了日時}}

### 1-2. ログ形式（例）

- 各発言を時系列で並べる：

```text
1. {{日時}} - {{話者}}（ID: {{ポストID}}）
   Content: {{本文}}
   Media: {{画像・リンクなど（あれば）}}
   Replied To: {{返信先ID or 引用元ID}}

2. {{…}}

…

```

※実運用では、必要に応じて要約済みログでもよい。

----------

## 2. 評価軸（7軸）

各軸は原則 **0〜100点** で評価する（⑦はペナルティ）。

----------

### ②-1. データ妥当性（Validity of Evidence）

**定義：**  
事実・統計・引用の正確さ、および根拠の提示の適切さ。

**主なチェックポイント：**

-   出典・統計・引用が明示されているか？
    
-   数値や割合など、検証可能なデータを用いているか？
    
-   データの読み方・解釈が適切か？（誤読・誤解釈がないか）
    
-   他者の提示したデータを正しく扱っているか？
    

**スコア目安：**

-   80–100：出典・数値・期間が明示され、読み方も正確。誤読なし。
    
-   60–79：一部にデータあり。大きな誤読はないが、裏付けは限定的。
    
-   40–59：主観メインで、データは断片的／曖昧。
    
-   20–39：データの誤読・曲解が目立つ。
    
-   0–19 ：事実に反する主張をベースにしている／デマに近い。
    

----------

### ②-2. 論理一貫性（Logical Consistency）

**定義：**  
主張と主張の間に矛盾がないか、論理の流れがスムーズか。

**チェックポイント：**

-   結論 → 根拠 →補足 の流れになっているか？
    
-   議論途中で主張が自家撞着していないか？
    
-   接続詞（したがって／一方で／しかし／つまり）で論理構造が示されているか？
    
-   主張を変えた場合、そのことに自覚的か？
    

**スコア目安：**

-   80–100：矛盾がほぼなく、論理の流れが明瞭。
    
-   60–79：基本は一貫しているが、一部揺れや曖昧さあり。
    
-   40–59：論点の飛躍・すり替えが見られる。
    
-   20–39：自分の過去発言と明確な矛盾あり。
    
-   0–19 ：そもそも論理になっていない／感情の羅列に近い。
    

----------

### ②-3. 引用の正確性（Interpretation Accuracy）

**定義：**  
相手の発言・提示データを正しく読み取り、歪めずに扱えているか。

**チェックポイント：**

-   相手の文章を正確に引用しているか？
    
-   意味の取り違え（誤読）がないか？
    
-   「Aの主張は◯◯だ」と要約する際に、意図を歪めていないか？
    
-   データの意味（例：◯◯のうち△△% vs △△の◯◯%）を正しく理解しているか？
    

**スコア目安：**

-   80–100：相手の発言・データを正確に引用し、誤読も即修正している。
    
-   60–79：基本的に正しいが、軽微なニュアンスずれあり。
    
-   40–59：誤読や曲解が目立つが、一部は正しく理解している。
    
-   20–39：相手の主張を意図的・継続的に歪めている印象。
    
-   0–19 ：議論の土台となる箇所で重大な誤読をしている。
    

----------

### ②-4. 再現性・明確性（Clarity / Reproducibility）

**定義：**  
第三者が読んで、同じ結論にたどり着ける程度に明確か。

**チェックポイント：**

-   曖昧語「普通」「常識」「みんな」が定義されているか？
    
-   範囲や対象（例：年齢／地域／属性）が示されているか？
    
-   誰が見ても検証可能な形で主張が表現されているか？
    

**スコア目安：**

-   80–100：定義・条件が明確で、再検証が容易。
    
-   60–79：おおよそ意味は分かるが、やや主観的。
    
-   40–59：曖昧語が多く、読み手によって解釈が分かれそう。
    
-   20–39：ほぼ印象論・感覚的表現のみ。
    
-   0–19 ：何を主張したいのか第三者から見て不明瞭。
    

----------

### ②-5. 説得力（Rhetorical Persuasiveness）

**定義：**  
論理とは別に、「読み手にとって説得的かどうか」という修辞的な力。

**チェックポイント：**

-   内容がコンパクトに整理されているか？
    
-   例や比喩が適切に使われているか？
    
-   論点を整理しながら話を進めているか？
    
-   無駄な煽りが少なく、主張そのものにフォーカスできるか？
    

**スコア目安：**

-   80–100：読者視点でも納得しやすい構成と語り口。
    
-   60–79：粗はあるが、要点は十分伝わる。
    
-   40–59：伝わるが、煽りや脱線で説得力が削がれている。
    
-   20–39：感情表現が主で、論点が伝わりにくい。
    
-   0–19 ：単なる罵倒とノリに終始している。
    

----------

### ②-6. 対話姿勢（Cooperative / Adversarial Stance）

**定義：**  
相手と対話する姿勢・誠実性・議論ルールの尊重度。

**チェックポイント：**

-   誤りを指摘されたときに修正できているか？
    
-   質問に答えようとする姿勢があるか？
    
-   レッテル貼りや決めつけに終始していないか？
    
-   一方的に「逃げた」「白旗」などの勝ち宣言で締めていないか？
    

**スコア目安：**

-   80–100：対立しつつもフェアな対話姿勢を保っている。
    
-   60–79：感情的な場面はあるが、基本は議論に向き合う。
    
-   40–59：煽りや罵倒が多く、対話は付随的。
    
-   20–39：議論よりもマウントが主目的に見える。
    
-   0–19 ：ほぼ荒らし・煽り行為に近い。
    

----------

### ②-7. 詭弁ペナルティ（Fallacy Load）

**定義：**  
詭弁・ごまかし・論理的に不誠実な手法の「量×重さ」をペナルティとして数値化。

**代表的な詭弁種別：**

-   人身攻撃（ad hominem）
    
-   藁人形論法（strawman）
    
-   論点すり替え・red herring
    
-   証明責任のすり替え（burden shifting）
    
-   スリッパリー・スロープ
    
-   cherry picking
    
-   whataboutism
    
-   過度な一般化（hasty generalization）
    

**スコアリング方法（例）：**

-   軽度の詭弁（1回）：+5 〜 +8
    
-   中程度（議論の流れを左右しうる）：+10 〜 +15
    
-   重度（議論の根幹を壊す）：+20 以上
    

**合計ペナルティ：**

-   0〜20：軽度（議論の全体構造は保たれている）
    
-   21〜40：中程度（詭弁にかなり依存）
    
-   41〜60：重度（まともな議論として成立しにくい）
    

※ **詭弁ペナルティは「減点」として扱う（値が大きいほど悪い）**

----------

## 3. スコアリングプロセス

### Step 1. ログの要約

1.  テーマ・論点の中核を一文〜三文で要約
    
2.  各側の主張の骨子（Claim）を整理
    
3.  重要なデータ・引用・定義を抽出
    

→ ここをテンプレで書く：

```markdown
### 3-1. 論点要約
- テーマ：{{一言でのテーマ}}
- Aサイドの主張骨子：{{要約}}
- Bサイドの主張骨子：{{要約}}

### 3-2. 重要なデータ／定義
- {{例：23%データの解釈、普通／一般の定義など}}

```

----------

### Step 2. 各軸の定性的評価 → スコア化

各軸について：

```markdown
#### {{軸名}}（{{Aサイド or Bサイド}}）

- 観察ポイント：
  - {{箇条書きで2〜4点}}
- 評価コメント：
  - {{短いコメント}}
- スコア（0〜100）：**{{数値}}**

```

----------

### Step 3. 最終スコア集計

#### 3-3-1. 加重合計（例）

必要に応じて、軸ごとに重みをつける：

```text
総合点 =
  0.20 * データ妥当性 +
  0.20 * 論理一貫性 +
  0.15 * 引用の正確性 +
  0.15 * 再現性・明確性 +
  0.15 * 説得力 +
  0.15 * 対話姿勢
  − 詭弁ペナルティ

```

※重みは用途に応じて調整可。
